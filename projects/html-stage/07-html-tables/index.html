How to create HTML tables

For creating a table we use the following HTML elements:

  • table: which defines the boundaries of the table
  • tr: to define a table row
  • th: to define table header
  • td: to define a generic table cell
The table accept a colspan and rowspan attributes that goes in table headers, which give us the ability to extend the table headers to cover multiple rows or columns.

Table Accessability

In order to make the HTML tables accessible, HTML provides multiple way, first, for insight users, we use CSS to make it easier to navigate through tables, and we use caption element to give a short presentation for the table, which is helpful for all type of users.

To make the table accessible by the screen readers, we have two ways, the first is to use the scope, which is an attribute that goes to table headers and tell wether that header is a header for a row or a column.
And another way using Ids and headers attributes, the latter is complete robust, and make no room for mistake but is require a lot of writing, and for that it is advisable to stick with the scope attribute unless necessary, for example when the table is complex and the scope is not enough.

HTML Images

Recall

We can embed an image using the img, it accept the attributes: src, alt, width, height. The src should point to the address where the image is stored, it could be an absolute or relative link, though relative links are preferred. the img name should be meaningful because the search-engine do read the img file name, which mean a good name is good for SEO.
The alt should contain a descriptive text to the images, incase if the image is failed to download, and also it is used for screen readers. and of course it is good for SEO.
It is a good practice to specify the width and height of the images so that the page content do not jump if the img did take some time to download which something happen often, specially if the img file is large, or the page contain numerous images, or the user connection is somewhat slow.
But specifying the width & height for the image is a complex topic when dealing with multiple screens, so it is an advanced topic that you may want to go back to in the future, but not for now.
